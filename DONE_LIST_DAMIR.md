## 2026-02-27 — Исправление алгоритма цены сведения

### Новый алгоритм расчёта цены аукциона
- Реализован расчёт через **кумулятивные объёмы** (нарастающий объём)
- Кумулятивный bid: сколько лотов готовы купить по цене X или выше
- Кумулятивный ask: сколько лотов готовы продать по цене X или ниже
- Цена сведения — точка пересечения кумулятивных объёмов

### Новые поля в ответе API
- `executed_lots` — лоты, которые **исполнятся** по цене аукциона
- `imbalance` — лоты, которые **НЕ исполнятся**
- `imbalance_direction` — `'bid'` (покупатели) или `'ask'` (продавцы)

### Увеличена глубина стакана
- `depth` увеличен с 10 до 50 уровней для более точного расчёта

### Объединены режимы в одну таблицу
- Убрана кнопка "Режим: Свечи / Стакан"
- Единая таблица: Актив, Цена 5м, Цена аукц., Откл. %, Лоты, Дисбаланс
- Дисбаланс с цветом: зелёный (покупатели), красный (продавцы)

## 2026-02-12 (поздний вечер) — Исправление проблем с кнопками

### Проблема
После исправления синтаксической ошибки кнопки «Настройки», «Режим: Свечи» и «Авто: ВКЛ» переставали работать после первого клика:
- Панель настроек не открывалась
- Режим не переключался
- Автообновление не запускалось
- Остальные кнопки переставали работать

### Причина
**Двойные обработчики событий:** у каждой кнопки было два обработчика:
1. Inline `onclick` в HTML
2. `addEventListener` в JavaScript

При клике срабатывали оба → панель открывалась и сразу закрывалась, режим переключался дважды.

### Решение
1. **Удалены дублирующие `addEventListener`** для `btnSettings`, `btnMode`, `btnAutoRefresh`
2. **Упрощены inline `onclick`** — вызов функций через `try/catch`
3. **Добавлены `try/catch`** в `toggleSettings()`, `toggleMode()`, `toggleAutoRefresh()`
4. **Добавлены проверки на `null`** для всех `getElementById`
5. **Панель настроек открывается синхронно**, список подгружается асинхронно

### Результат
- ✅ Панель настроек открывается один раз
- ✅ Режим переключается один раз
- ✅ Автообновление переключается один раз
- ✅ Остальные кнопки продолжают работать
- ✅ Ошибки логируются в консоль, но не ломают интерфейс

**Коммит:** `58d1de9` — "fix: Настройки/Режим/Авто — один обработчик, try/catch, панель открывается сразу"

## После теста аукциона — правки

1. **Переключение на 5 сек во время аукциона**  
   Время считалось по локальному времени браузера. Исправлено: московское время считается из UTC (UTC+3). После каждого обновления интервал пересчитывается, поэтому при наступлении аукциона переключение на 5 сек происходит вовремя.

2. **Отклонение в %**  
   В режиме стакана использовалась «цена закрытия» из ответа стакана. Отклонение унифицировано: везде считается от **вчерашней цены закрытия** (из дневных свечей) до цены аукциона. В `/api/orderbook` добавлена подстановка вчерашнего close из свечей.

3. **Точность цены**  
   Цена аукциона округляется до 2 знаков после запятой (копейки).

4. **Режим «Свечи»**  
   Данные те же: цена аукциона и отклонение считаются так же. Раньше во время аукциона обновление оставалось раз в 30 мин из‑за бага с временем; после исправления п.1 оба режима обновляются каждые 5 сек в окне аукциона.

5. **Лоты**  
   Объёмы берутся из API стакана (quantity по уровням). Если в терминале показываются другие цифры, возможно API отдаёт только **видимую часть айсберг-заявок**; полный объём по ним в API может быть недоступен. В коде добавлен комментарий об этом.
