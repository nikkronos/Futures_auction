### Альтернатива: gRPC стримы (для снятия лимита)

**Проблема:** REST API имеет лимит 600 запросов/мин на сервис котировок. При интервале 2 сек и 18 инструментах мы на пределе.

**Решение:** Перейти на gRPC стримы (подписка на данные в реальном времени).

**Преимущества gRPC:**
- Подписка на 300 инструментов в одном соединении
- Данные приходят в реальном времени (без polling)
- Нет лимита на количество запросов
- Интервал обновления: 100 мс (ограничение API)

**Недостатки:**
- Требует переписать архитектуру (WebSocket на фронте, gRPC на бэке)
- Сложнее в реализации и отладке
- Нужна библиотека gRPC для Python

**Документация:** https://russianinvestments.github.io/investAPI/limits/

### Отложено / Не планируется
- [ ] Встраивание в терминал Т-Инвестиции — **невозможно** (закрытый продукт, нет SDK для виджетов)
