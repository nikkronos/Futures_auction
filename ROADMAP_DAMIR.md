### Альтернатива: gRPC стримы (для снятия лимита)

**Проблема:** REST API имеет лимит 600 запросов/мин на сервис котировок. При интервале 2 сек и 18 инструментах мы на пределе.

**Решение:** Перейти на gRPC стримы (подписка на данные в реальном времени).

**Преимущества gRPC:**
- Подписка на 300 инструментов в одном соединении
- Данные приходят в реальном времени (без polling)
- Нет лимита на количество запросов
- Интервал обновления: 100 мс (ограничение API)

**Недостатки:**
- Требует переписать архитектуру (WebSocket на фронте, gRPC на бэке)
- Сложнее в реализации и отладке
- Нужна библиотека gRPC для Python

**Документация:** https://russianinvestments.github.io/investAPI/limits/

### Важно, но не сделано (зафиксировано для следующих итераций)
- [ ] **1 сек обновление, до 50 инструментов** — отдельная задача; нужен gRPC или существенная переработка серверного кэша.
- [ ] **Серверное хранение профилей** — чтобы один URL давал те же настройки с любого устройства (сейчас только localStorage).
- [ ] **Документ процесса бэкапов** — формальное описание в `docs/`: когда делать бэкап, какие команды (git + копии на сервере).
- [ ] **Режим симуляции аукциона** — тестирование без привязки к реальному окну 8:50–9:00.
- [ ] **TradingView как источник свечей** — публичного API нет; источник остаётся T-Invest.
- [ ] **Отклонение/5м для TTF и др.** — при сохранении расхождений с брокером рассмотреть fallback (например last price) для отдельных инструментов.

- [ ] Встраивание в терминал Т-Инвестиции — **невозможно** (закрытый продукт, нет SDK для виджетов)
